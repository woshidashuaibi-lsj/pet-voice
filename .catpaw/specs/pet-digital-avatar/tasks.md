# Live2D 宠物分身 MVP - 实现任务清单

## 整体时间规划

第 1 周（Day 1-5）：数据层设计 + 基础页面

第 3 周（Day 11-15）：完善 + 小游戏 + 测试

## 任务列表

### 第 1 周：数据层 + 基础页面

- [ ] **Task 1.1：创建 `pet_avatar_states` 云数据库集合**
  - 在微信云数据库中创建 `pet_avatar_states` 集合
  - 定义数据模型（参考设计文档 1.1 节）
  - 设置权限规则：用户只能读写自己宠物的数据
  - 创建索引（petId, userId, updatedAt）
  - _需求: 1.0_

- [ ] **Task 1.2：编写 `updateAvatarState` 云函数**
  - 创建云函数 `cloudfunctions/updateAvatarState/index.js`
  - 实现四维数据计算逻辑（contentment / energy / affection / health）
  - 实现进化等级判断逻辑
  - 处理首次初始化（新宠物）和定期更新两种场景
  - 添加错误处理和日志
  - _需求: 1.1, 1.2_

- [ ] **Task 1.3：编写 `recordInteraction` 云函数**
  - 创建云函数 `cloudfunctions/recordInteraction/index.js`
  - 实现交互记录逻辑（异步，不阻塞前端）
  - 更新相关维度值
  - 计算连续活跃天数
  - _需求: 5.0_

- [x] **Task 1.4：新建 `pages/avatar/index` 页面骨架**
  - 创建页面文件：index.wxml / index.js / index.wxss
  - 设计页面布局（参考设计文档 5.1 节）：
    - 顶部导航栏
    - 中央模型容器（id="live2d-container"）
    - 四维数据条区域
    - 进化等级显示
    - 底部功能按钮
  - 在 app.json 中注册页面
  - _需求: 6.0_

- [x] **Task 1.5：实现四维数据条 UI 组件**
  - 创建四个数据条，分别显示 contentment / energy / affection / health
  - 每个数据条包括：
    - 图标（❤️ / ⚡ / 🏥 / 😸）
    - 数值（0-100）
    - 进度条（平滑动画）
    - 颜色编码（绿色 > 60 / 黄色 30-60 / 红色 < 30）
  - 实现过渡动画（300ms）
  - _需求: 6.0_

### 第 2 周：Live2D 集成 + 互动系统

- [ ] **Task 2.1：获取 Live2D 模型资源**
  - 从开源库或官方资源获取可用的二次元宠物 Live2D 模型（.moc3 / .json）
  - 确保模型文件大小 ≤ 2MB
  - 放入 `miniprogram/assets/models/` 目录
  - 验证模型格式和完整性
  - _需求: 2.0_

- [ ] **Task 2.2：集成 Live2D 库到小程序**
  - 下载 `live2d.min.js` 或通过 npm 获取
  - 在小程序中配置 npm 包（wxss 中引入）
  - 在 `pages/avatar/index.js` 中初始化 Live2D
  - 创建 Live2D 渲染实例
  - 测试基础动作播放（idle / happy / curious 各一个）
  - _需求: 2.0, 2.1_

- [x] **Task 2.3：实现动作库管理系统**
  - 在 `pages/avatar/utils/animations.js` 中定义 15 个动作（参考设计 2.2）
  - 每个动作包括：name / duration / loop / sound / emotionalBoost / dialog
  - 实现动作播放方法 `playAnimation(animationId)`
  - 实现随机动作选择方法 `getRandomAnimation(type)`
  - 实现动作队列系统（支持连续播放多个动作）
  - _需求: 2.0, 2.1_

- [ ] **Task 2.4：实现页面初始化和欢迎动作**
  - 页面 onLoad 时：
    - 从云数据库读取 avatar_state 数据
    - 如果不存在则调用 updateAvatarState 云函数创建
    - 初始化 Live2D 模型
    - 播放欢迎动作（happy / curious / playful 随机）
  - 处理加载状态和错误提示
  - _需求: 1.1, 2.1_

- [ ] **Task 2.5：实现点击热区检测系统**
  - 在 Live2D 容器上添加触摸事件监听器
  - 实现点击区域判断函数：
    - 获取点击坐标 (x, y)
    - 根据宠物模型边界框判断属于 head / back / body / tail / 其他
    - 计算点击精度（中心点更准确）
  - 实现双击检测（两次点击间隔 < 300ms）
  - 实现长按检测（按住 > 3 秒）
  - _需求: 3.0_

- [x] **Task 2.6：实现点击响应逻辑和数据反馈**
  - 创建交互映射表（CLICK_INTERACTION）
  - 实现 `handleInteraction(area)` 方法：
    - 获取对应的响应配置
    - 根据性格和当前维度调整概率和响应
    - 播放对应动作
    - 显示对话气泡
    - 更新前端维度数据
    - 显示 +3 affection 等浮起数字
  - 实现冷却机制（不同区域 300-600ms）
  - 调用 recordInteraction 云函数记录交互
  - _需求: 3.0, 4.0_

- [x] **Task 2.7：实现智能响应机制**
  - 创建 `getAdjustedResponse(baseResponse, petState, personality)` 函数
  - 根据性格调整响应概率
  - 根据能量值和健康度调整动作选择
  - 处理夜间睡眠状态（22:00-08:00）
  - 处理低能量/低健康度情况（显示疲倦/难受动作）
  - 返回调整后的响应配置
  - _需求: 3.0, 4.0_

- [ ] **Task 2.8：实现对话气泡系统**
  - 创建对话库（`utils/dialogs.js`），包含不同互动类型的对话文案
  - 实现 `showDialog(text)` 方法：
    - 在宠物上方显示气泡
    - 显示 2 秒后自动消失
    - 支持多条气泡队列
  - 文案示例：
    - tap_head: ["你摸我头干嘛呢？", "这样挠舒服~"]
    - double_tap: ["主人最好了！"]
    - etc.
  - _需求: 3.0_

### 第 3 周：完善 + 小游戏 + 测试

- [ ] **Task 3.1：实现情绪到表现的映射**
  - 实现 moodScore 计算函数：
    ```
    moodScore = contentment * 0.3 + energy * 0.2 + affection * 0.3 + health * 0.2
    ```
  - 根据 moodScore 映射到情绪段级（very_sad / sad / neutral / happy / very_happy）
  - 实现相应的视觉变化：
    - 皮肤色调变化（灰暗 → 明亮）
    - 眼神状态变化（无神 → 闪闪发光）
    - 尾巴状态变化（下垂 → 快速摇晃）
  - 应用到 Live2D 模型（通过参数调整或 shader）
  - _需求: 4.0_

- [ ] **Task 3.2：实现升级系统和视觉反馈**
  - 监听进化等级变化
  - 当等级升级时：
    - 播放"胜利"动作
    - 显示升级动画（金光闪闪、烟花效果）
    - 弹出升级提示文案
    - 显示新解锁的动作
  - 记录升级时间戳
  - _需求: 4.0, 6.0_

- [ ] **Task 3.3：实现第一个小游戏 - 投喂游戏**
  - 创建新页面 `pages/avatar-game-feed/index`
  - 游戏机制：
    - 显示随机的食物图标
    - 用户快速点击以投喂正确的食物
    - 根据宠物的饮食偏好（从 pets.memory 读取）判断是否正确
    - 倒计时 30 秒，计算得分
  - 游戏结束后：
    - 计算分数和奖励（满足度 +5-10）
    - 显示宠物进食动画
    - 返回分身页面前更新 avatar_state
  - _需求: 3.0 的扩展_

- [ ] **Task 3.4：完善页面交互和细节**
  - 页面返回按钮实现
  - 分享功能（分享宠物分身卡片）
  - 加载状态处理
  - 网络错误提示和重试
  - 进度条和数据条的平滑动画优化
  - _需求: 6.0_

- [ ] **Task 3.5：性能优化和兼容性测试**
  - 测试 Live2D 渲染性能（FPS）
  - 内存泄漏检测（长时间运行）
  - 不同手机型号测试（iOS / Android）
  - 低端手机适配（帧率优化、模型简化）
  - 网络延迟处理
  - _需求: 8.0_

- [ ] **Task 3.6：集成现有功能联动**
  - 完成对话后，avatar_state 能正确更新
  - 完成打卡后，宠物分身页面实时反馈
  - 宠物档案页能跳转到分身页
  - AI 对话页完成后，记忆被正确提取并影响分身性格标签
  - _需求: 7.0_

- [ ] **Task 3.7：完整流程测试 + Bug 修复**
  - 从小程序首页进入宠物分身页
  - 完成多次点击交互（验证维度更新）

  - 返回首页，再次进入，验证数据持久化

## 风险点和对策

| 风险 | 对策 |
|------|------|
| Live2D 库加载失败 | 提前测试库的兼容性，准备 polyfill 或替代方案 |
| 点击热区判断不准 | 事先调试点击坐标映射，可视化热区便于调试 |
| 四维数据计算逻辑复杂 | 先用固定值测试 UI，再逐步完善数据公式 |
| 性能问题（帧率低） | 定期检查 GPU 占用，考虑降低动画复杂度 |
| 数据不一致 | 严格测试云函数逻辑，确保幂等性 |
